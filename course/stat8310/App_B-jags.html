8  Appendix: Introduction to JAGS and BUGS language

8.0.1 Why learn JAGS?

So far, we have written Monte Carlo and Gibbs samplers directly in R.
This is useful for understanding the mechanics of Bayesian computation, but it becomes inconvenient when models are more complicated.

JAGS stands for Just Another Gibbs Sampler.
It is a program for fitting Bayesian hierarchical models using MCMC, especially Gibbs sampling and related methods.

JAGS uses the BUGS language, where BUGS stands for Bayesian inference Using Gibbs Sampling.

The basic idea is simple:

  1. we describe the statistical model in BUGS syntax;
  2. we provide the observed data;
  3. we choose which parameters to monitor;
  4. JAGS runs the MCMC algorithm and returns posterior samples.

8.1 What BUGS language does

BUGS language is a model specification language.
It is not a general programming language like R or Python.

Its role is to describe:

  • the sampling model,
  • the prior distributions,
  • any deterministic relationships among parameters,
  • optional derived quantities.

For example, instead of writing the Gibbs updates by hand, we simply write the model:

\[ Y_i \mid \theta \sim \text{Poisson}(\theta), \qquad \theta \sim \text{Gamma}(a,b), \]

and JAGS handles the sampling automatically.

8.2 Basic structure of a JAGS model

A JAGS model is usually written inside a model { ... } block.

model {
  # likelihood
  for (i in 1:n) {
    y[i] ~ dpois(theta)
  }

  # prior
  theta ~ dgamma(a, b)
}

This contains two parts:

  • the likelihood: y[i] ~ dpois(theta)
  • the prior: theta ~ dgamma(a, b)

It is not a general programming language like R or Python.

Its role is to describe:

  • the sampling model
  • the prior distributions

  • optional derived quantities

For example:

Instead of writing the Gibbs sampler ourselves, we simply describe the model and JAGS handles the sampling.

9.3 Basic structure of a JAGS model

A JAGS model is written inside a model {} block.

model {
  for (i in 1:n) {
    y[i] ~ dpois(theta)
  }

  theta ~ dgamma(a, b)
}

The first part describes the likelihood, and the second part specifies the prior.

9.4 BUGS syntax rules

9.4.1 Stochastic nodes

Random variables are written using ~.

theta ~ dgamma(a, b)
y[i] ~ dpois(theta)

9.4.2 Deterministic nodes

Deterministic relationships use <-.

sigma <- 1 / sqrt(tau)
mu[i] <- alpha + beta * x[i]

9.4.3 Loops

Repeated data are handled with loops.

for (i in 1:n) {
  y[i] ~ dnorm(mu, tau)
}

9.4.4 Comments

Use # for comments.

# prior distribution
theta ~ dnorm(0, 0.01)

9.5 Precision vs variance

JAGS uses precision rather than variance for normal distributions.

dnorm(mean, precision)

Precision is

\[ \tau = \frac{1}{\sigma^2}. \]

Example:

tau <- 1 / sigma^2

9.6 Common distributions

Distribution JAGS syntax
Normal dnorm(mu, tau)
Poisson dpois(lambda)
Binomial dbin(p, n)
Bernoulli dbern(p)
Gamma dgamma(a, b)
Beta dbeta(a, b)
Uniform dunif(a, b)

9.7 Example: Beta–Binomial model

\[ Y \mid \theta \sim \text{Binomial}(n,\theta), \qquad \theta \sim \text{Beta}(a,b) \]

JAGS model:

model {
  y ~ dbin(theta, n)
  theta ~ dbeta(a, b)
}

For multiple observations:

model {
  for (i in 1:n) {
    y[i] ~ dbern(theta)
  }
  theta ~ dbeta(a, b)
}

9.8 Example: Poisson–Gamma model

\[ Y_i \mid \theta \sim \text{Poisson}(\theta), \qquad \theta \sim \text{Gamma}(a,b) \]

JAGS:

model {
  for (i in 1:n) {
    y[i] ~ dpois(theta)
  }

  theta ~ dgamma(a, b)
}

9.9 Example: Normal model

\[ Y_i \mid \theta,\tau \sim \text{Normal}(\theta,\tau) \]

Priors

\[ \theta \sim \text{Normal}(\mu_0,\tau_0), \qquad \tau \sim \text{Gamma}(a,b) \]

JAGS model:

model {
  for (i in 1:n) {
    y[i] ~ dnorm(theta, tau)
  }

  theta ~ dnorm(mu0, tau0)
  tau ~ dgamma(a, b)

  sigma <- 1 / sqrt(tau)
}

9.10 Data list in R

data_jags <- list(
  y = y,
  n = length(y),
  a = 2,
  b = 1
)

9.11 Parameters to monitor

params <- c("theta")

9.12 Example JAGS workflow

library(jagsUI)

fit <- jags(
  data = data_jags,
  parameters.to.save = params,
  model.file = textConnection(model_string),
  n.chains = 3,
  n.iter = 5000,
  n.burnin = 1000
)

print(fit)

9.13 Initial values

Optional but sometimes useful.

inits <- function() {
  list(theta = 1)
}

9.14 Posterior output

Typical outputs include

  • posterior means
  • credible intervals
  • trace plots
  • density plots
  • effective sample size
  • \(\hat{R}\) convergence diagnostic

9.15 Useful BUGS functions

Function Meaning
log(x) logarithm
exp(x) exponential
sqrt(x) square root
pow(x,a) power
step(x) indicator function

Example:

tau <- pow(sigma, -2)

9.16 Indexing

for (i in 1:n) {
  y[i] ~ dnorm(mu[i], tau)
  mu[i] <- alpha + beta * x[i]
}

9.17 Common beginner mistakes

  1. Forgetting that normal uses precision
  2. Mixing deterministic <- and stochastic ~
  3. Assuming all R syntax works inside JAGS
  4. Forgetting to pass constants in the data list

9.18 Summary

  • JAGS performs Bayesian inference using MCMC.
  • Models are written in the BUGS language.
  • ~ denotes a distribution.
  • <- denotes deterministic relationships.
  • Normal distributions use precision instead of variance.
  • JAGS separates model, data, and parameters.
